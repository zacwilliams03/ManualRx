# Client PDF Download + PDF Redesign

**Date:** 2026-06-09
**Status:** Approved for implementation

---

## Goal

1. Let clients download PDFs of their prescribed sessions directly from the client Dashboard.
2. Redesign all three PDF components (PrescriptionPDF, AllSessionsPDF, ProgramPDF) with a polished tabular layout informed by ui-ux-pro-max healthcare design guidelines.

---

## Design Decisions (all confirmed via visual mockups)

| Decision | Choice |
|---|---|
| PDF layout | Tabular — exercises as rows |
| Per-set handling | Indented sub-rows beneath the exercise name row |
| Seconds measurement | Fixed "REPS / SEC" column header; unit shown inline in cell ("6 reps", "45 sec") |
| Program structure | Week badge + horizontal rule → session with left border → exercise table |
| Client download trigger | Download icon on each Dashboard session card + "Download All" button |
| Client PDF content | Prescription only (exercise plan, not completed log) |

---

## PDF Visual Design (applies to all three components)

### Colours
Use the existing app brand tokens — do **not** introduce new shades:
- `NAVY = '#1E2D3D'` — primary text
- `TEAL = '#29B5CC'` — accent, table header background, exercise number
- `TEAL_LIGHT = '#E8F9FC'` — meta strip background, per-set row tint (**updated from `#E1F5FA`** — apply consistently across all three components)
- `TEAL_BORDER = '#A8E6F0'` — meta strip border, row separators (**new constant**; replaces the old `BORDER = '#D4E8E8'` in all three components — retire `BORDER` entirely)
- `GREY = '#6B7280'` — secondary text (notes, bodyweight, dashes)
- `WHITE = '#FFFFFF'` — table header text

### Page layout
- `paddingTop/Bottom: 48`, `paddingHorizontal: 48` (unchanged)
- Footer: `position: absolute, bottom: 24` — "Generated by ManualRx" left, "Page X" right

### Header block (all PDFs)
```
┌──────────────────────────────────────────────────────┐
│  ManualRx  (TEAL accent on "Rx")    CLINIC NAME      │
│  EXERCISE PROGRAM (8px, grey, 2px letterSpacing)     │
│  ────── 2.5px TEAL border bottom ─────────────────── │
```

### Meta strip (single-session / PrescriptionPDF)
A teal-tinted pill row below the header:
```
[ Client: Jane Smith ]  [ Session: A — Hamstring ]  [ Frequency: 3× / week ]
```
- Background `TEAL_LIGHT`, border `TEAL_BORDER`, borderRadius 6, padding 8×12
- Label: 7.5px TEAL uppercase; Value: 9.5px NAVY bold

### Meta strip (AllSessionsPDF)
Same pill style but shows only `Client` and `Date` — no single session name since multiple sessions are listed:
```
[ Client: Jane Smith ]  [ Date: 9 Jun 2026 ]
```
`Date` is the **generation date** (today), same as the existing component — not a prescription date.

### Program meta (ProgramPDF only)
- Programme name as a large (15px bold) title
- Subtitle: "Client: X · Start: D · Duration: N weeks" in 8px grey

### Exercise table structure
Each session renders one table with these columns:

| Column | Width | Notes |
|---|---|---|
| # | 22pt | TEAL bold, exercise index |
| EXERCISE | flex | Name bold NAVY; notes italic 7px grey on second line |
| SETS | 30pt | Centred, bold NAVY |
| REPS / SEC | 36pt | Centred; shows "10 reps" or "45 sec" with unit as smaller grey suffix |
| WEIGHT | 46pt | Centred; "BW" in grey, kg values in bold NAVY |
| TEMPO | 50pt | Monospace pill: background TEAL_LIGHT, border TEAL_BORDER, text TEAL |

**Table header row:** `backgroundColor: TEAL`, all text white, 7px bold uppercase, 0.5pt letterSpacing.

**Normal exercise rows:** alternating white / very-light-teal (`#FAFEFF`), 1pt `TEAL_BORDER` bottom border.

### Per-set rows (when `prescription_exercise_sets.length > 0`)
1. **Name row:** # cell | EXERCISE cell (name + "PER-SET" badge) | SETS blank | REPS/SEC blank | WEIGHT blank | **TEMPO cell** (badge if tempo set, "—" in grey if not). The name row does NOT colspan — it keeps all six cells so tempo is always visible.
2. **Sub-header row:** blank | "Set / Reps / Weight" labels (6.5pt grey uppercase) filling EXERCISE + SETS + REPS/SEC + WEIGHT cells | TEMPO blank. Background `#F0FDFF`.
3. **Set rows:** blank # | set number (TEAL bold) in EXERCISE column | reps (NAVY bold) in REPS/SEC | weight in WEIGHT | TEMPO: empty `<View style={{ width: 50 }} />` (must be present to maintain column alignment). Background `#F0FDFF`.
4. All sub-rows share a 1pt `TEAL_BORDER` bottom only on the last set row.
5. Notes (if any) appear as a teal left-border box in the EXERCISE column below the last sub-row.
6. Sub-header row TEMPO cell: also an empty `<View style={{ width: 50 }} />` — same reason.

### Notes
- Rendered as a `View` with `borderLeftWidth: 2, borderLeftColor: TEAL, backgroundColor: TEAL_LIGHT, padding: 6, borderRadius: 4, marginTop: 4`.
- Text: 9px NAVY.
- If no notes, the notes view is omitted entirely.

### Tempo badge
```
background: TEAL_LIGHT
border: 1pt TEAL_BORDER
borderRadius: 4
padding: 2×5
fontFamily: Courier (monospace available in react-pdf)
fontSize: 7.5pt
fontWeight: bold
color: TEAL
```
Format: `E-B-C-T` (e.g. "3-1-2-0"). If no tempo, render "—" in grey.

### Tempo footnote
If any exercise in the document has tempo, render at the bottom of the last page:
> `* Tempo (seconds): Eccentric — Bottom Pause — Concentric — Top Pause`

## ProgramPDF — Week/Session structure

```
┌─ Week 1 ─────────────────────────────────────────────┐   ← TEAL badge + full-width rule

  │ Session A — Hamstring Strength  ·  3× per week      ← 3pt left border (TEAL)
  │ [exercise table]

  │ Session B — Hip Stability  ·  2× per week           ← 3pt left border (TEAL)
  │ [exercise table]

┌─ Week 2 ──────────────────────────────────────────────┐
  ...
```

- **Week badge:** `backgroundColor: TEAL, color: white, 8px bold uppercase, padding 4×10, borderRadius 4`.
- **Week rule:** `flex: 1, height: 1, backgroundColor: TEAL_BORDER` — sits to the right of the badge in a flex row.
- **Session left border:** `borderLeftWidth: 3, borderLeftColor: TEAL`. Session name 10px NAVY bold, frequency label 7.5px TEAL.
- Sessions within a week separated by `marginBottom: 10`.
- Weeks separated by `marginBottom: 18`.

## Client PDF Download — Dashboard

### Where it lives
- **Per-card:** small download icon button on each active session card in `src/pages/client/Dashboard.jsx`.
- **Download All:** a secondary button in the page hero/header area (below the page title). Only shown when the client has ≥ 1 active session.

### Header data — no extra queries needed
Dashboard already has everything the PDF header requires:

| PDF prop | Source | Notes |
|---|---|---|
| `clinicName` | `useClinicName()` — already called in Dashboard | Works for clients via `clients.therapist_id` join |
| `clientName` | `profile.name` from `useAuth()` | `profile` is `{ id, email, role, name, theme }` — use `profile.name` |
| `weightUnit` | `useWeightUnit()` — add to Dashboard | For clients reads from `clients.weight_unit` |
| `therapistId` | `session.therapist_id` from the already-loaded `sessions` state | Present in Dashboard's existing select |

**No additional Supabase round-trips are needed for the PDF header.**

### On-demand exercise fetch (triggered on button click)
```js
// Single session — prescriptionId from the card, therapistId not needed for this query
supabase
  .from('prescription_exercises')
  .select('sets, reps, weight, therapist_notes, measurement_type, bilateral,
           tempo_eccentric, tempo_bottom_pause, tempo_concentric, tempo_top_pause,
           prescription_exercise_sets(set_number, reps, weight),
           exercises(name)')
  .eq('prescription_id', prescriptionId)
  .order('id', { ascending: true })
```

### Download All
Fetch exercises for every active prescription in `sessions` state (iterate over them), then pass to `AllSessionsPDF`. Mirror the therapist's `downloadAllPDF` pattern in `Prescribe.jsx`.

### Loading state
Track a `downloadingId` state: `null` when idle, a prescription ID when a single download is in progress, or `'all'` when the download-all is in progress. The relevant button shows a spinner and is disabled while its ID matches `downloadingId`.

### Error handling
Track a `downloadError` state string. On fetch or PDF render failure, set a brief message ("Failed to download PDF"). Render it as a small red text below the "Download All" button (or inline on the card for per-card failures). Auto-clear after 5 seconds via `setTimeout`.

### Filename
- Single: `{session.name}.pdf` (spaces replaced with hyphens, lowercased)
- All: `{profile.name}-all-sessions.pdf`

| Test | Key exercise props |

## Constraints

- `@react-pdf/renderer` has no HTML `<table>` element — all table rows are `<View flexDirection="row">` with fixed-width child `<View>` cells.
- Available monospace font: `Courier` (built-in). Used for tempo badges.
- No Google Fonts — stick to Helvetica, Helvetica-Bold, Courier.
- `borderRadius` on `<View>` is supported.
- Per `@react-pdf/renderer` docs: `position: absolute` works for the footer; page breaks handled automatically.
