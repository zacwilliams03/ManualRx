# Export All Sessions PDF

**Date:** 2026-05-27
**Status:** Approved

## Overview

Make the "Export PDF" button on the Prescribe page hero functional. Clicking it generates a single PDF containing all active prescribed sessions for the current client — each showing the exercise list and completion frequency — and triggers a browser download.

## Scope

- **In scope:** `AllSessionsPDF` React PDF component; wiring the placeholder button in `Prescribe.jsx`
- **Out of scope:** Session history/completion stats, inactive prescriptions, PDF from the Clients list page

## Architecture

### New file: `src/components/therapist/AllSessionsPDF.jsx`

A `@react-pdf/renderer` Document component. Props:

```
{
  clinicName: string,
  clientName: string,
  prescriptions: Array<{
    name: string,
    frequencyLabel: string,     // e.g. "Daily", "Every 3 days", "Weekly", "No repeat"
    exercises: Array<{
      name: string,
      sets: number,
      reps: number,
      weight: number | null,
      therapist_notes: string | null,
    }>
  }>
}
```

**PDF layout (single Page, A4, continuous flow):**

1. Header — same branding as `PrescriptionPDF`: ManualRx logo bar, clinic name subtitle, client name + date meta row.
2. For each prescription:
   - Section heading: prescription name (bold, 13pt, NAVY) + frequency label (9pt, TEAL) on the same row.
   - Exercise list: name (bold), sets × reps (+ weight or "Bodyweight"), therapist notes box (teal background).
   - Thin horizontal divider after each prescription except the last.
3. Footer: "Generated by ManualRx" centered at bottom.

Reuses the same color constants (NAVY, TEAL, TEAL_LIGHT, GREY, BORDER) and `weightDisplay` from `pdfUtils`.

### Changes to `src/pages/therapist/Prescribe.jsx`

**New state:**
```js
const [allPdfLoading, setAllPdfLoading] = useState(false)
const [allPdfError, setAllPdfError] = useState(false)
```

**New `downloadAllPDF()` function:**
1. Derive active prescription IDs from the already-loaded `sessions` state using the existing `isActive` helper.
2. If none are active, return early (button is hidden anyway).
3. Set `allPdfLoading = true`, clear `allPdfError`.
4. Single Supabase query: `prescription_exercises` with `.in('prescription_id', activeIds)`, selecting `prescription_id, sets, reps, weight, therapist_notes, exercises(name)`, ordered by `created_at asc`.
5. Group results by `prescription_id`, map to the shape `AllSessionsPDF` expects — preserving the order of `activeSessions` (sorted active-first by the existing `sortedSessions` logic).
6. Call `pdf(<AllSessionsPDF .../>).toBlob()`, trigger download as `{sanitise(clientName)}-all-sessions.pdf`.
7. `finally`: set `allPdfLoading = false`.
8. On error: set `allPdfError = true`, log to console.

**Button wiring (existing placeholder at line ~344):**
- `onClick` → `downloadAllPDF`
- `disabled` when `allPdfLoading`
- Label: `"Exporting…"` while loading, `"Export PDF"` otherwise
- Hidden (`display: none`) when no active sessions exist
- Small error label below button when `allPdfError` is true

## Data flow

```
User clicks "Export PDF"
  → downloadAllPDF()
    → filter sessions → activeIds
    → Supabase: prescription_exercises .in(activeIds)
    → group by prescription_id
    → pdf(<AllSessionsPDF/>).toBlob()
    → URL.createObjectURL → <a>.click() → download
    → URL.revokeObjectURL
```

## Error handling

- Query error: catch block sets `allPdfError = true`, shows inline error label next to button.
- Empty active sessions: button is hidden; `downloadAllPDF` returns early if called anyway.

## Testing

- At least one active prescription with exercises → PDF downloads, contains all active sessions.
- Mix of active and inactive prescriptions → only active appear in PDF.
- Active prescription with no exercises → section renders with heading but empty exercise list (no crash).
- All prescriptions inactive → "Export PDF" button is hidden.
- Supabase query failure → error label appears, loading spinner stops.
